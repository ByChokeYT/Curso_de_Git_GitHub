## 🔬 Estándares de Industria (Workflow Models)

### Gitflow (Modelos Formales)
Ideal para productos con versiones de lanzamiento (Releases).
-   `main`: Producción.
-   `develop`: Integración.
-   `feature/*`: Desarrollo de funciones.

### Trunk-Based Development (Modelos Ágiles)
Ideal para despliegue continuo. Todos trabajan en una sola rama (`main`) con cambios pequeños (Short-lived branches). Requiere una suite de tests automáticos extremadamente robusta.

## ⚙️ El Proyecto Final: El Hito de Maestría

Para completar tu formación, debes ser capaz de ejecutar este flujo sin errores:

1.  **Explorar:** Clonar un Monorepo y realizar un `Sparse Checkout` de un componente.
2.  **Desarrollar:** Crear una rama, implementar una mejora y realizar un **Semantic Commit**.
3.  **Asegurar:** Firmar el commit con GPG.
4.  **Colaborar:** Abrir un Pull Request y pasar el pipeline de **GitHub Actions** (Linting + Build).
5.  **Desplegar:** Merge con estrategia de `Squash` para mantener el historial limpio.
